# L1

rateyourdj-l1 schema 在终端打印 L1 接受的字段结构和类型约束，不创建文件

rateyourdj-l1 init demo-user 创建用户 ID 为 **demo-user** 的空画像

rateyourdj-l1 validate profile_patch.json 检查后续模块传入的字典是否合法

rateyourdj-l1 import demo-user profile_patch.json 检查字典能否正确合并并保存

完整自动化检查: python -m unittest discover -s tests -v

# L2

## 结构

重新注册命令：

```
python -m pip install -e .
```

查看 schema 和创建空歌曲画像：

```
rateyourdj-l2 schema
rateyourdj-l2 init song-001
```

生成位置：

```
data/song_profiles/song-001.json
```

支持 **validate** 和 **import**，暂未包含 API 采集、标签推断或 embedding 生成。L1+L2 共 22 个测试全部通过。

## 采集

在[Spotify Developer Dashboard](https://developer.spotify.com/dashboard)中create app，Redirect URI为http://127.0.0.1:8888/callback，获取Client Secret和ID，随后cd到目录，保存到环境

```
export SPOTIFY_CLIENT_ID="你的Client ID"
export SPOTIFY_CLIENT_SECRET="你的Client Secret"
```

确认配置成功

```
test -n "$SPOTIFY_CLIENT_ID" && echo "Client ID configured"
test -n "$SPOTIFY_CLIENT_SECRET" && echo "Client Secret configured"
```

最后运行测试

```
RUN_LIVE_API_TESTS=1 python -m unittest \
  tests.test_data_collection_smoke.SpotifyCollectionSmokeTest -v
```

可采集到的字段有：

* **title**
* **artist/artists**
* **album**
* **release\_year**
* **duration\_ms**
* **song\_id**：可使用 **spotify\_track\_id**
* **data\_source**

在[Last.fm API Account](https://www.last.fm/api/account/create) 申请 API Key，保存到环境

```
export LASTFM_API_KEY="你的API Key"
```

查看结果

```
RUN_LIVE_API_TESTS=1 python -m unittest \
  tests.test_data_collection_smoke.LastfmCollectionSmokeTest -v
```

可采集到的字段有：

* **track\_tags**：歌曲级原始标签及 count
* **artist\_tags**：歌手级原始标签及 count
* **normalized\_track\_tags**：归一化到 **0–1** 的歌曲标签
* **candidate\_genres**：

Last.fm 返回的是社区标签，不会明确区分 genre、mood、年代或场景，后续必须做标签分类。后续为在Last.fm中收集原始 tags，保存 source_tags，随后进行标签清洗和分类，最后生成 aligned_features.genres
